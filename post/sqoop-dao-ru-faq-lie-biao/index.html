在使用sqoop从oracle导入到hive，报错：sqoop could not load db driver class:oracle.jdbc.OracleDriver

在安装sqoop server的节点上放oracle的驱动包

/opt/cloudera/parcels/CDH/lib/sqoop/lib/ojdbc6.jar

并且修改权限为777

进入 $JAVA_HOME/jre/lib/security 目录下的 java.security 文件

cd $JAVA_HOME/jre/lib/security

vim java.security
修改 securerandom.source=file:/dev/../dev/urandom，如果这个参数不管用改为 securerandom.source=file:///dev/urandom 或 手动添加参数-D mapred.child.java.opts=``"-Djava.security.egd=file:/dev/../dev/urandom"

如果依旧有问题进行以下操作

export HADOOP_OPTS=-Djava.security.egd=file:/dev/../dev/urandomsqoop import -D mapred.child.java.opts="-Djava.security.egd=file:/dev/../dev/urandom"

先在命令行或脚本设置HADOOP_OPTS，然后再执行sqoop import就可以了，最好加上mapred.child.java.opts参数,并且在第二次导入的时候一定要加上--hive-overwrite参数。
注：有时候-m需要设置成1才可以。刚开始我将-m设置成2报错，我设置成1就可以了。

在脚本中export…和sqoop import -D mapred.child.java.opts都是不能少的。
